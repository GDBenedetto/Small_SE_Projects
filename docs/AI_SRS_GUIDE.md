# AI SRS Guide

## 🎯 Purpose
This document defines how an AI should create a **System Requirements Specification (SRS)**.  
It follows best practices from Systems Engineering (INCOSE, IEEE 29148, ISO/IEC/IEEE 15288).

---

## 🧠 Core Principles

- Use **clear, concise, and precise sentences**
- Use **"shall"** for all requirements
- Ensure **requirements are testable**
- Separate:
  - Functional
  - Behavioral
  - Non-functional
- Use **consistent terminology**
