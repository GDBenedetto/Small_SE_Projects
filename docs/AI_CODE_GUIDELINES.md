# AI Code Guidelines

## 🎯 Purpose
This document defines how an AI should generate **clean, readable, and maintainable code**.

---

## 🧠 Core Principles

- Write **simple and clear code**
- Prefer **readability over cleverness**
- Use **consistent formatting**
- Avoid unnecessary complexity
- Write code that is easy to debug

---

## 🧩 Code Structure

- Separate:
  - Initialization (setup)
  - Main logic (loop / main)
  - Helper functions
- Use meaningful sections

---

## 🏷️ Naming Rules

- Use descriptive names
- Avoid abbreviations
- Use English only

### Example

---

## 💬 Comments

- Write **short and precise comments**
- Explain **why**, not only **what**
